Units 4-6

Vocab

1. Necessary Conditions:


  • Mutual Exclusion: At least one resource must be held in a non-shareable mode.
  • Hold and Wait: Processes currently holding resources may request additional resources while waiting for others to release theirs.
  • No Preemption: Resources cannot forcibly be taken from a process.
  • Circular Wait: There must be a circular chain of two or more processes, each waiting for a resource held by the next process in the chain.

2. Resource Allocation Graph:


Definition: A graphical representation of the allocation of resources in an OS and the relationship between processes and resources.

  • Nodes:
    • Processes (represented as circles).
    • Resources (represented as squares).
  • Edges:
    • Request Edge (r): Represents a process requesting a resource.
    • Assignment Edge: Represents a resource being allocated to a process.

3. Methods for Handling Deadlocks:


  • Deadlock Prevention: Ensures that at least one of the necessary conditions for deadlock cannot hold.
  • Deadlock Avoidance:
    • Allows the system to allocate resources only if it can guarantee that the system will remain in a safe state.
    • Safe State: A state in which the system can allocate resources to each process in some order and still avoid a deadlock.
    • Banker's Algorithm: Checks if the system can allocate requested resources without entering an unsafe state.
  • Deadlock Detection: Used if a system does not employ either a deadlock prevention or a deadlock avoidance algorithm, allowing deadlocks to occur.
    • Wait-for Graph: Represents threads waiting for other threads to release resources they need.
      • Derived from the resource allocation graph by removing resource nodes and collapsing appropriate edges.
    • Benefits: Allows for detecting deadlocks and determining better resource allocation strategies.
  • Deadlock Recovery: The process of resolving deadlocks that have already occurred in a system. It involves breaking the deadlock and restoring system functionality.
    • Steps to Deadlock Recovery:
      • Process Termination: Abort one or more processes to break the circular wait.
      • Resource Preemption: Temporarily take resources from some processes and give them to others.
      • Rollback: Rollback one or more processes to a previous safe state.

Additional Important Points:


  • Understanding Deadlocks: Deadlocks are a significant problem in concurrent systems where multiple processes or threads compete for limited resources. They can cause the system to halt and require intervention to resolve.
  • Practical Implications: In real-world systems, deadlock prevention and avoidance techniques are crucial to ensure system reliability and performance. Effective deadlock detection and recovery mechanisms are necessary to handle unexpected deadlock situations gracefully.
